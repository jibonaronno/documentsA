ESP32-S3-Touch-LCD-1.46-Test — ESP-IDF v6.0.1 Port: Change Summary

A complete record of every modification made while porting this Waveshare demo from its original ESP-IDF version to ESP-IDF v6.0.1 (xtensa-esp-elf-gcc 15.2.0, target esp32s3, picolibc, LVGL 8.3.11).

1. Build-system / dependency fixes (IDF v6.0 breaking changes)

1.1 json component removed → replaced with managed cjson

File: main/idf_component.yml

  • Added managed dependency: 
    espressif/cjson: "~1.7.19"

File: components/espressif__esp-sr/CMakeLists.txt

  • Replaced json with cjson in the REQUIRES lists for the esp32, esp32s3 and esp32p4 blocks.

1.2 driver/i2s.h split out → esp_driver_i2s

File: components/chmorgan__esp-audio-player/CMakeLists.txt

  • Added esp_driver_i2s to the component's REQUIRES line.

1.3 GCC 15 no longer transitively includes <cmath> in some C++ TUs

Files:

  • components/espressif__esp-dsp/modules/kalman/ekf/common/ekf.cpp
  • components/espressif__esp-dsp/modules/kalman/ekf/common/ekf_imu13states.cpp

#include <cmath>

(Earlier esp32 target caused GPIO_NUM_47/48 undeclared errors.)

2. newlib → picolibc ABI shim

picolibc exposes a 256-entry _ctype_b[256] instead of newlib's 257-entry _ctype_[257]. The pre-built esp-sr libraries reference the old symbol.

2.1 New file NEW

File: main/newlib_ctype_compat.c

  • Defines unsigned char _ctype_[257] and uses a constructor to populate entries 1..256 from picolibc's _ctype_b[256]. Entry [0] is the sentinel (0).

2.2 main/CMakeLists.txt

  • Added ./newlib_ctype_compat.c to SRCS.
  • Appended at the end to force-link the shim: 
    target_link_libraries(${COMPONENT_LIB} INTERFACE "-u _ctype_")

3. MultiNet disabled (prebuilt lib ABI-incompatible with IDF v6.0)

Cause: libmultinet.a was built against newlib + older heap APIs and crashes with StoreProhibited at address 0x8 on first detect call.

3.1 sdkconfig.defaults

  • # CONFIG_USE_MULTINET is not set
  • CONFIG_SR_MN_EN_NONE=y
  • Commented out CONFIG_SR_MN_CHINESE_SPEECH_COMMAND_MN5_QUANT8
  • Added CONFIG_SPI_MASTER_ISR_IN_IRAM=y

3.2 main/MIC_Driver/MIC_Speech.c

  • Removed #include "esp_mn_iface.h" and #include "esp_mn_models.h".
  • Changed i2s_init(i2s_port_t …) → i2s_init(int …) (the typedef was removed in v6.0).
  • Rewrote detect_hander (sic) to wake-word-only: no MultiNet create / detect / destroy calls.

Wake-word ("Hi ESP") detection still works.

5. LVGL driver — tearing fix + DMA backpressure

5.1 main/LVGL_Driver/LVGL_Driver.h

  • LVGL_BUF_LEN changed from (EXAMPLE_LCD_WIDTH * EXAMPLE_LCD_HEIGHT / 40) → (EXAMPLE_LCD_WIDTH * EXAMPLE_LCD_HEIGHT) (full screen).
  • Declared new public callback: 
    bool lvgl_notify_flush_ready(esp_lcd_panel_io_handle_t panel_io,
                             esp_lcd_panel_io_event_data_t *edata,
                             void *user_ctx);

5.2 main/LVGL_Driver/LVGL_Driver.c

Added includes:

#include "driver/gpio.h"
#include "freertos/FreeRTOS.h"
#include "freertos/semphr.h"
#include "Display_SPD2010.h"

Tear-effect (TE) GPIO sync — GPIO 18 from panel:

  • static SemaphoreHandle_t s_te_sem
  • te_gpio_isr_handler (IRAM_ATTR, gives sem from ISR on rising edge)
  • te_sync_init(): configures GPIO 18 input + posedge interrupt, installs ISR service with ESP_INTR_FLAG_IRAM, attaches handler.
  • Called from LVGL_Init() right after lv_disp_drv_register(&disp_drv).

Flush callback — example_lvgl_flush_cb:

  • Drains any stale TE pulse, then xSemaphoreTake(s_te_sem, pdMS_TO_TICKS(20)) to wait for the next V-blank.
  • Calls esp_lcd_panel_draw_bitmap(...).
  • No longer calls lv_disp_flush_ready() synchronously — completion is signalled from the SPI ISR via lvgl_notify_flush_ready.

New lvgl_notify_flush_ready:

  • Called by the panel-IO driver on on_color_trans_done.
  • Guards against firing during the boot self-test (disp == NULL).
  • Calls lv_disp_flush_ready(drv).

New example_lvgl_wait_cb:

  • Registered on disp_drv.wait_cb.
  • Calls vTaskDelay(1) so that LVGL's busy-wait loop in refr_area_part yields and the IDLE0 task can feed the WDT.

LVGL_Init() changes:

lv_color_t *buf1 = heap_caps_malloc(LVGL_BUF_LEN * sizeof(lv_color_t),
                                    MALLOC_CAP_SPIRAM);                   // was MALLOC_CAP_DMA | MALLOC_CAP_INTERNAL
lv_disp_draw_buf_init(&disp_buf, buf1, NULL, LVGL_BUF_LEN);               // single full-frame buffer, no buf2
...
disp_drv.full_refresh = 1;          // entire frame in one flush_cb call
disp_drv.wait_cb     = example_lvgl_wait_cb;
disp = lv_disp_drv_register(&disp_drv);
te_sync_init();                     // attach TE GPIO ISR after disp exists

8. Final working configuration

  • Target: esp32s3, octal PSRAM @ 80 MHz
  • QSPI LCD clock: 20 MHz (quad lanes ≈ 10 MB/s — sustainable from PSRAM)
  • LVGL buffer: single full-frame, 412 × 412 × 2 ≈ 340 KB in PSRAM
  • Refresh model: full_refresh = 1, gated on panel TE rising edge (GPIO 18), flush_ready deferred to SPI DMA completion
  • Voice: Wake-word "Hi ESP" works; MultiNet command recognition disabled
  • Frame rate: ~25 fps (340 KB / 10 MB/s ≈ 34 ms per frame), tear-free
