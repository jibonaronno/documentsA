Bootloader can pass various information to kernel after kernel image has been loaded and when jumping to kernel entry point. The book is calling it Bootloader to Kernel interface. The options are:
1. Argument passing to the kernel.
 Bootloader can pass argument to the kernel in a specific form. The kernel can take the argument for itself or pass it to appropriate module/driver. After system start-up, the argument can be seen from cat /proc/cmdline command.
