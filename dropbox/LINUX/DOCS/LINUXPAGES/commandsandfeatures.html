Enabling Virtual Box Shared Folder for Arch Linux.
Share a Directory from windows drive for virtual box arch linux require some commands. Though it can be automated somehow. We are using some command line operations to access the shared folder. In my case my shared folder name is Shared .
For me I needed to apply following commands:
1. sudo modprobe -a vboxguest vboxsf vboxvideo.
2. sudo mount -t vboxsf shared /media/shared.
Now the virtual box assigned shared folder is available under /media/shared . So here we had to load modules modules and then mount the shared folder through the module/modules. Here vboxsf (Virtual Box Shared Folder) module may be used for acces the shared folder.

SSH & SAMBA. Using SSH.

There are some factors about SSH and SAMBA that needs to be memorised for regular user. When to connect from a SSH client, we need to provide the username. Then the client asks for password. The user name should be created on the Linux system as well.
sudo useradd -c "comment" -g adm -m newuser
sudo passwd newuser
You need to edit /etc/sudoers file to enable the newusers privilage to execute sudo commands. Though you can SSH to the system as root user or any other admin user, and can change files. Add following lines to sudoers file inside any where:
newuser ALL=(ALL:ALL) ALL

But in lubuntu 15.04 we had to use the command adduser jibon. Then it requires a password to create the new user. Then we apply another command visudo. It is actually an env variable to open /etc/sudoers file to add the user privilages.

Now use following command to get access to remote windows machine. 

		pi@raspberrypi ~ $ smbclient -U aronno //192.168.12.20/E
		Enter aronno's password:
		session request to 192.168.12.20 failed (Called name not present)
		Domain=[ARONNO-PC] OS=[Windows 7 Professional 7600] Server=[Windows 7 Professional 6.1]
		smb: \>


Use ls command to get file/folder list. Use cd commands. Type help to egt list for commands available by smbclient. Use get command to get any file from host.
