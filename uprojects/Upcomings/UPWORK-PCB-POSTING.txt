OVERVIEW
--------
We're building an outdoor/industrial air-quality monitoring system. Field units ("nodes")
sample multiple gas, particulate, and environmental sensors, relay readings to a master node
over a LoRa mesh, and the master uploads to the cloud over cellular. The firmware (ESP-IDF C
on ESP32-S3) and the cloud platform are already built and running in the field.

We have an existing, semi-working V1 board that serves as a reference, and we need an experienced
hardware engineer to design V2. V2 is a substantial change, not a minor revision: the main goals
are to fix all of the power issues and add the missing connections, then complete and optimize
the layout and deliver manufacturing-ready files. Treat V1 as a reference for what the node does
and what's currently wired (it runs, but with known power problems and gaps) - V2 is where the
real design work happens.

This is an ongoing hourly relationship: we expect revisions, additional sensor variants, and
continued board spins over time, so we're looking for someone we can keep working with.


THE DEVICE
----------
The node is built around an off-the-shelf LilyGo T3-S3 module (ESP32-S3 + onboard LoRa radio).
The board we need designed is a CARRIER / BASEBOARD that the T3-S3 plugs into and that breaks
its GPIO out to our sensors, the cellular modem, and the power system. The MCU, LoRa radio +
antenna, USB-C / USB-UART, and 18650 charging are already on the T3-S3 - so this is integration,
analog front-end, and power work, NOT a from-scratch ESP32-S3 / RF design.

Powered by a 3.6 V Li-SOCl2 primary battery pack (EVE ER34615 D-cell, ~19,000 mAh) paired with an
HPC1530 hybrid layer capacitor (HLC) for pulse current. Solar is an optional input connector.
Two build variants on a common board:

- Slave node  - samples sensors, talks LoRa only (via the T3-S3 radio).
- Master node - same sensing, plus a cellular modem (Quectel BG95) driven over UART/AT commands
                for the cloud uplink.

ALREADY PROVIDED by the LilyGo T3-S3 (off-the-shelf, not your design):
- ESP32-S3 MCU
- LoRa radio + antenna ([[confirm variant - e.g. SX1262, 915 MHz US band]])
- USB-C, USB-to-UART
- 18650 holder + charging (present on the module, but NOT the main power path - the node runs
  from the external ER34615 primary pack)

WHAT THE CARRIER BOARD MUST ADD (your design):
- Gas sensors: SPEC Sensors electrochemical cells - CO, NO2, SO2, mercaptan (TBM), IAQ - each
              needs a proper analog front-end (potentiostat / transimpedance amp -> ADC) with
              low-noise layout
- CO2 + temp + humidity: Sensirion SCD41 (I2C)
- Particulates: Plantower PMS9103M PM1/PM2.5/PM10 sensor (UART) - slave variant
- Cellular (master variant only): Quectel BG95 + SIM + antenna, UART/AT; power design for
              LTE/NB-IoT TX current bursts
- Pressure:   [[sensor part - confirm, I2C]]
- Power:      3.6 V Li-SOCl2 primary pack (EVE ER34615, ~19 Ah) + HPC1530 hybrid layer capacitor
              (HLC) to source the BG95 pulse currents the high-impedance Li-SOCl2 cell cannot;
              optional solar input connector; rail regulation (buck/LDO) for sensors and modem;
              ADC-based battery (and solar) voltage/current monitoring; very low quiescent current
              for multi-year battery life; current budgeting for deep-sleep + cellular TX bursts
- Integration: T3-S3 header/footprint, GPIO breakout to all peripherals, connectors, status LEDs
- A GPIO / pin-allocation map will be provided to the selected freelancer.
