Dot Row Storage Architecture

1. Wire Format (incoming packet)

ByteFieldValue
0start 0xA5 (delimeter)
1cmdID 0x05 = CMD_DOT_ROW
2length 108 or 216 (bytes of pixel data)
(3...N)payload packet bit array of dots

Dots are packed 8 per byte -- 1728 dots = 216 bytes; 864 dots = 108 bytes

2. In-memory queue structure

Once parsed, each row is stored in a circular buffer defined in PrinterQueue.c:

    typedef struct {
        uint8_t data[PRINTER_QUEUE_ROW_SIZE]; // 216 bytes max
        uint16_t len;                          // actual byte count (108 or 216)
    } PrinterRow_t;

    static PrinterRow_t queue[PRINTER_QUEUE_MAX_ROWS]; // 40 slots
    static volatile uint16_t head = 0;
    static volatile uint16_t tail = 0;

So the queue holds up to 40 rows, each up to 216 bytes (1728 dots / 8), for a total of ~8.6 KB of RAM.

Key design points

  • IRQ safety: __disable_irq() / __enable_irq() guards every queue access since AddRow is called from the USB IRQ context and DequeueRow from the printer task/timer.
  • Overflow policy: If the queue is full (nextHead == tail), the incoming row is silently dropped (with a nop placeholder for a breakpoint).
  • Queue watermarks defined in PrinterQueue.h: QUEUE_ALMOST_EMPTY (5), QUEUE_ALMOST_FULL (38), QUEUE_READY_TO_START (35) — used for flow-control responses back to the host.

SPI1 is used to shift dot data into the thermal head's shift register. Here's what the code confirms: In Printer.c:

How it works:

  • A dequeued row (108 or 216 bytes) is pushed out over SPI1 TX via DMA to clock the bits into the thermal head's internal shift register.
  • Once the data is fully shifted in, the strobe signals (STROBE_0–3, driven via GPIO through TIM16 ISR in Strobes.c) are fired sequentially to heat the selected dot groups and burn the image onto the paper.
  • The motor then advances the paper by one line pitch, and the process repeats every ~2 ms.
So the full print pipeline per row is: SPI1 DMA (shift data) → Strobe GPIOs (heat) → Motor step (advance paper).

Observer that the function if(PrinterQueue_DequeueRowPtr(&pData)) Dequeues a row from the printer queue if available.

Serial data of each row is transferred to the TPH head via SPI1 DMA then PA6 pin latches the output. Now Strobe signals are fired sequentially to heat the selected dot groups and burn the image onto the paper. Strobes are driven via GPIO through TIM16 ISR in Strobes.c.

The strobe heating time is defined by the darkness setting, darkness is set by command CMD_SET_DARKNESS.


Command List In Firmware

File: PrinterProtocol.c

// --- Called once full packet is ready ---
static void Protocol_HandlePacket(uint8_t *packet, uint8_t cmd, uint8_t len) {
    switch (cmd) {
    case CMD_SET_PROPERTIES:
        if (len != 2) break;
        PrinterProtocol_HandleSetProperties(packet, len);
        break;

    case CMD_QUERY_PAPER_STATUS:
        localPrinterStatus = PrinterReadStatus(NULL);
        PrinterProtocol_SendPaperStatus((PaperStatus_t)localPrinterStatus.paperPresent);
        break;

    case CMD_SET_DARKNESS:
        if (len != 1) break;
        PrinterSetDarkness((int)packet[0]);
        break;

    case CMD_SET_SPEED:
        if (len != 1) break;
        PrinterSetSpeed((PrintSpeed_t)packet[0]);
        break;

    case CMD_DOT_ROW:
        PrinterProtocol_HandlePrintDotRow(packet, len);
        break;

    case CMD_FEED_OUT:
        if (len != 1) break;
        PrinterFeed((int)packet[0]);
        break;

    default:
        // Unknown command — ignore
        break;
    }
}
